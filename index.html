Methodology

The first step in the project was for the entire team to participate in brainstorming ideas for improving the existing site. This involved reviewing the questionnaire answers which were provided by Alistair, discussing the purpose(s) of the website, and identifying items for improvement. The team was also motivated by Alistair’s remarks in class that this project was an opportunity to “bring it all together”, i.e. to showcase all of the skills that we had learned during COMP 1950. Out of this process came a vision for a new site, outlined in the following section.

Bringing It Together

As mentioned previously, it was important to TeamOne that their work on this project showcase the skills they have learned by taking COMP1950. The following list outlines how and where these skills have been employed:

  • Open proto-site in a new window. target="_blank" not supported in HTML 5, so us javascript?
  • file/folder names all lowercase, no spaces, etc (week 1)
  • semantic markup (week 2) separate code from style, logical styles
  • SEO: use distinct titles, meta descriptions, etc (week 3)
  • use HTML5 elements appropriately (week 3)
  • CSS: two media for assignment - screen and print (week 4, week 5), proper use of selectors, etc. CSS should validate correctly.
  • CSS: use a reset (normalize?).(week 5)
  • CSS: media queries: (week 5) Not sure if we want to do this, or have separate css files for each media
  • SSI: use SSI to include common code in proto-site: header(s) footer, navigation (week 6)
  • CMS/Templates: we are using Github for code-control, and the Blog example from Tumblr (week 7, week 8)
  • Methodology: client site has a page to discuss rationale, related to methodology (week 9)
