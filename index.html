Methodology

The website development methodology taught in COMP 1950 recommends using the following project phases: Definition, Site Development, Design, Testing and Launch. It became clear during the Definition phase of this project, that this standard methodology would need to be modified slightly. The team understood that the required end-product of this project is a "client site", a "proto-site" and a blog, from which the client (Alistair) may then choose features to implement later. A fully-featured production site, with content for all possible lecture pages, was not a requirement. Therefore, the Site-Development, Design, Testing and Launch phases for this project refer only to activities related to the client site, the proto-site and the blog.

There were several features of the existing COMP 1950 site which TeamOne saw no need to change, such as links to Alistair's email address, and related links: BCIT, my.BCIT, etc. The team decided to not duplicate these features on the proto-site; they could easily be added later if the proto-site is taken into production.

The first step in the Site Development phase was for the entire team to participate in brainstorming ideas for improving the existing site. This involved reviewing the questionnaire answers which were provided by Alistair, discussing the purpose(s) of the website, and identifying items for improvement. The team was also motivated by Alistair's remarks in class that this project was an opportunity to "bring it all together", i.e. to showcase all of the skills that we had learned during COMP 1950. Out of this process came a vision for a new site, outlined in the Rationale section of this client site.

The project manager, Rod, divided the Design and Testing phases into smaller tasks, for which teams of two volunteered to take responsibility. A Dropbox folder was used for miscellaneous documents such as draft content, to-do lists and the task breakdown structure. A Git repository was used extensively to organize code development.

Bringing It Together

It was important to TeamOne that their work on this project showcase the skills they have learned by taking COMP1950. The following list outlines how and where these skills have been employed:

  • file/folder names all lowercase, no spaces, etc (week 1)
  • semantic markup used extensively to separate code from style, logical styles (week 2)
  • content optimized for SEO: use distinct titles, meta descriptions, and ensure that 404 errors are handled with a 404.html page (week 3). [Note: server limitations prevented the team from enabling this functionality in the proto-site.]
  • used HTML5 elements appropriately (week 3)
  • CSS: two media for assignment - screen and print (week 4, week 5), proper use of selectors, etc. CSS should validate correctly.
  • CSS: use a reset (normalize.css)(week 5)
  • SSI: use SSI to include common code in proto-site: header(s) footer, navigation (week 6)
  • CMS/Templates: we are using Github for code-control, and rolling out a blog using Tumblr (week 7, week 8)
  • Methodology: Disciplined methodology used, and this client site includes a section which discusses methodology (week 9)
  • Javascript and JQuery: Accordion-style navigation menu on proto-site uses JQuery (week 10)

During the course of the project, members of the team were introduced to some new tools and resources. In particular Slickmap (used for producing the site map) and Balsamiq (used to create the wireframe diagrams) were found to be excellent resources. Ideally we would have used Google Custom Search for for the search functionality, but FreeFind gave us more control over the site reindexing.
